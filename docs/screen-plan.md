# Tabular Review — UI/UX 화면기획서 (v0.1)

> 작성일: 2026-06-04
> 범위: Admin / 일반 유저 화면군의 정보구조(IA) + 화면별 레이아웃·구성요소·인터랙션
> 제외: 로그인/인증/권한 라우팅 (역할은 진입 시 결정되어 있다고 가정), 챗 고도화(tool-calling·RAG)

---

## 1. 제품 개요

비정형 문서(계약서, 약관 등)를 **도메인=문서종류 단위의 워크스페이스**로 관리한다.

- **Admin**: 워크스페이스(도메인)를 선택해 문서를 업로드(insert)하고, 추출 컬럼을 정의·관리하며, 추출(Run Analysis)을 실행하고 결과를 검증한다. → 기존 Tabular Review 그리드 화면이 거의 그대로 여기에 해당.
- **일반 유저**: **챗 화면을 메인**으로, 자연어로 문서를 검색하거나 문서 기반 질문을 한다. 답변의 근거를 **읽기전용 그리드/문서 뷰어**로 확인한다. (추출·컬럼편집 불가)

### 사용자 역할 요약

| | Admin | 일반 유저 |
|---|---|---|
| 워크스페이스 생성/삭제 | ✅ | ❌ |
| 문서 업로드/삭제 | ✅ | ❌ |
| 컬럼 정의/편집 | ✅ | ❌ |
| 추출 실행(Run) | ✅ | ❌ |
| 그리드 보기 | ✅ 편집 가능 | ✅ **읽기전용** |
| 셀 검증/편집 | ✅ | ❌ (보기만) |
| 챗 | ✅ | ✅ **메인** |
| 문서 검색 | ✅ | ✅ |

## 2. 정보구조 (IA) — 평면 구조

워크스페이스 하나 = **하나의 도메인 = 하나의 문서종류**. (예: "계약서", "약관", "소송서면")
워크스페이스 안에 문서 N개 × 컬럼 M개의 Tabular Review 그리드가 있다.

```
앱
├─ [Admin 모드]
│   ├─ A-0  워크스페이스 목록 (도메인 카드)
│   └─ A-1  워크스페이스 상세 = Tabular Review 그리드
│            ├─ A-2  컬럼 관리 (추가/편집/라이브러리)  ── 패널/모달
│            ├─ A-3  문서 업로드/관리                 ── 드롭존/패널
│            └─ A-4  검증 사이드바 (셀 인용 확인)       ── 우측 사이드바
│
└─ [User 모드]
    ├─ U-0  챗 메인 (도메인 선택 + 대화)            ← 일반 유저 진입점
    ├─ U-1  검색 결과 (읽기전용 그리드)             ── 챗에서 전환/병렬
    └─ U-2  문서 뷰어 / 인용 확인                   ── 사이드바/오버레이
```

> 평면 구조이므로 **좌측 레일에서 워크스페이스(도메인)를 전환**하는 것이 핵심 내비게이션이다. 별도 "문서종류" 하위 단계는 없다.

## 3. 글로벌 레이아웃

두 모드 공통으로 **좌측 레일(워크스페이스 전환) + 중앙 작업영역 + 우측 사이드바(컨텍스트)** 3분할을 기본 골격으로 한다.

```
┌──────┬───────────────────────────────────────────────┬──────────────┐
│      │  Top Bar (워크스페이스명 / 액션 / 모드별 버튼)      │              │
│ 좌측 ├───────────────────────────────────────────────┤  우측         │
│ 레일 │                                               │  사이드바      │
│      │            중앙 작업영역                         │  (검증/챗/    │
│ (WS  │     (그리드 또는 챗 또는 검색결과)                 │   문서뷰어)   │
│ 전환)│                                               │              │
└──────┴───────────────────────────────────────────────┴──────────────┘
```

- **좌측 레일**: 워크스페이스(도메인) 목록 아이콘/이름. 현재 선택 강조. Admin은 하단에 "＋ 새 워크스페이스".
- **우측 사이드바**: 현재 `App.tsx`의 애니메이션 사이드바 컨테이너 재사용 (none/verify/chat, 400px↔900px 확장).

## 4. ADMIN 화면군

### A-0. 워크스페이스 목록 (도메인 대시보드)

Admin 진입 시 첫 화면. 도메인=문서종류 카드 그리드.

```
┌──────┬────────────────────────────────────────────────────────────┐
│  ▦   │  Workspaces                              [＋ 새 워크스페이스]  │
│  WS  │  ──────────────────────────────────────────────────────────  │
│ 레일 │  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│      │  │ 📄 계약서       │ │ 📄 약관        │ │ 📄 소송서면     │         │
│ ▦계약│  │ 문서 128 · 컬럼 9│ │ 문서 54 · 컬럼 6│ │ 문서 12 · 컬럼 4│         │
│ ▦약관│  │ 최종수정 6/2    │ │ 최종수정 5/28  │ │ 최종수정 5/19  │         │
│ ▦소송│  │ [열기] [⚙][🗑] │ │ [열기] [⚙][🗑] │ │ [열기] [⚙][🗑] │         │
│  ＋  │  └──────────────┘ └──────────────┘ └──────────────┘         │
└──────┴────────────────────────────────────────────────────────────┘
```

- **카드 구성요소**: 도메인 아이콘/이름, 문서 수, 컬럼 수, 최종수정일, 진행상태(예: 추출 완료율 바).
- **액션**: `열기`(→ A-1), `⚙ 설정`(이름/문서종류·아이콘 변경), `🗑 삭제`(확인 모달).
- **＋ 새 워크스페이스**: 모달 — 이름, 문서종류 라벨, (옵션) 컬럼 템플릿 프리셋 선택.
- **빈 상태**: "아직 워크스페이스가 없습니다. 첫 도메인을 만들어 문서를 추가하세요." + CTA.

### A-1. 워크스페이스 상세 = Tabular Review 그리드

**기존 `App.tsx` + `DataGrid.tsx`가 거의 그대로 대응.** 좌측 레일만 추가되는 형태.

```
┌──────┬───────────────────────────────────────────────────────┬─────────┐
│  ▦   │ 계약서  ▸ Untitled Project   [Chat][Export][⊞Wrap]      │         │
│  WS  │                       [＋문서][모델▾][▶ Run Analysis]    │  우측    │
│ 레일 ├───────────────────────────────────────────────────────┤ 사이드바 │
│      │ ☐│ 문서명        │ 지배법    │ 계약일    │ MFN조항  │ ＋   │ (verify │
│ ▦계약│ ─┼──────────────┼─────────┼─────────┼────────┼────  │  /none) │
│ ▦약관│ ☐│ ACME_MSA.pdf │ 뉴욕주 ●  │ 2024-01 ●│ 있음 ⚠  │      │         │
│ ▦소송│ ☐│ Beta_NDA.pdf │ 델라웨어●  │ 2023-11 ●│ 없음 ●  │      │         │
│      │ ☐│ ...          │ extracting…(스피너)            │      │         │
│  ＋  │                                                       │         │
└──────┴───────────────────────────────────────────────────────┴─────────┘
```

- **상단 바**: 워크스페이스명(도메인) + 프로젝트명(인라인 편집), 우측에 액션군 — `Chat`, `Export CSV`, `Wrap`, `＋문서`, `모델 선택`, `Run Analysis`(또는 선택 행 있으면 `Re-run (N)`, 처리중이면 `Stop`).
- **그리드(중앙)**:
  - 행 = 문서, 열 = 컬럼. 좌측 체크박스로 행 선택(선택 재실행용).
  - 셀 = 추출값 + 신뢰도 점(High/Med/Low 색상) + 상태 아이콘(✓검증됨 / ⚠Low확인필요).
  - 열 헤더: 컬럼명 + 타입, 헤더 클릭 → 편집(A-2), 우측 끝 `＋`로 컬럼 추가.
  - 셀 클릭 → 우측 검증 사이드바(A-4). 문서명 클릭 → 문서 뷰어.
  - 드래그&드롭으로 문서 추가(현 `onDropFiles`).
- **상태 표시**: 컬럼 추출 중 스피너, 변환 중 전체 오버레이("Converting Documents — Docling").
- **기존 코드 매핑**: 이 화면 = 현재 `App.tsx`의 메인 + `DataGrid.tsx` 거의 그대로. 추가 작업은 좌측 레일과 "워크스페이스 컨텍스트"로 상태를 감싸는 것.

### A-2. 컬럼 관리 (추가 / 편집 / 라이브러리)

기존 `AddColumnMenu.tsx` + `ColumnLibrary.tsx` 대응. 그리드 위 팝오버/모달.

```
컬럼 추가/편집 팝오버                      컬럼 라이브러리 모달
┌─────────────────────────┐            ┌──────────────────────────────┐
│ 컬럼 편집            [라이브러리]│            │ Column Library        [Import]│
│ 이름  [ 지배법          ]│            │ 카테고리: [전체][법률][재무][날짜]│
│ 타입  [text ▾]          │            │ ┌───────────────┐            │
│ 프롬프트                  │            │ │ 지배법 (text)    │ [＋추가]   │
│ [ What is the governing │            │ │ "What is the..." │           │
│   law of this contract?]│            │ ├───────────────┤            │
│        [✨ AI로 프롬프트 생성]│            │ │ 계약 총액 (number)│ [＋추가]   │
│ [삭제]          [취소][저장]│            │ └───────────────┘            │
└─────────────────────────┘            └──────────────────────────────┘
```

- **컬럼 편집**: 이름 / 타입(text·number·date·boolean·list) / 자연어 프롬프트 / `AI 프롬프트 생성`(`generatePromptHelper`) / 삭제.
- **라이브러리**: 저장된 템플릿을 카테고리별로 탐색 → 클릭 추가. Import/Export(localStorage 기반, 기존 `fileStorage.ts`).
- **Admin 전용** — 일반 유저에게는 컬럼 헤더의 `＋`/편집 진입점 자체를 숨긴다.

### A-3. 문서 업로드 / 관리

`＋문서` 또는 드롭존. 기존 `DocumentUpload.tsx` / `documentProcessor.ts`.

```
┌───────────────────────────────────────────┐
│  문서 추가 — 계약서 워크스페이스                  │
│  ┌─────────────────────────────────────┐  │
│  │   📎 여기에 PDF·DOCX·TXT를 끌어다 놓거나     │  │
│  │        클릭하여 선택하세요                  │  │
│  └─────────────────────────────────────┘  │
│  변환 중: ACME_MSA.pdf  ▓▓▓▓▓░░ (Docling)   │
│  ✅ Beta_NDA.pdf                            │
└───────────────────────────────────────────┘
```

- 업로드 → Docling 변환(`/convert`) → 마크다운으로 그리드에 행 추가.
- 변환 진행/실패 표시(현재는 전체 오버레이 + alert → **문서별 진행/실패 행**으로 개선 권장).
- 문서 관리: 그리드 행에서 제거(`handleRemoveDoc`), 다중 선택 → 선택 재실행.

### A-4. 검증 사이드바 (셀 인용 확인)

기존 `VerificationSidebar.tsx`. 셀 클릭 시 우측에서 확장.

```
┌─────────────────────────────────┐
│ 지배법                       [확장][✕]│
│ ─────────────────────────────── │
│ 값:  뉴욕주 (New York)            │
│ 신뢰도: ● High                   │
│ 추론:  계약서 18.2조에 명시…         │
│ 출처:  p.12  [원문에서 보기]        │
│ ─────────────────────────────── │
│ …governed by the laws of the    │
│ State of [New York]…(하이라이트)   │  ← 확장 시 원문 전체 + <mark>
│ ─────────────────────────────── │
│           [✓ 검증 완료]            │
└─────────────────────────────────┘
```

- 좁게(400px): 값/신뢰도/추론/출처 요약. 넓게(900px): 원문 전체에서 인용구 노란 하이라이트 + 자동 스크롤.
- `검증 완료` → 셀 상태 `verified`(✓). **이 검증 액션은 Admin 전용.**

## 5. USER 화면군 (일반 유저)

### U-0. 챗 메인 (진입점)

일반 유저의 기본 화면. 좌측에서 **도메인(워크스페이스)을 선택**하고 가운데서 대화한다.

```
┌──────────┬──────────────────────────────────────────────┬───────────┐
│ 도메인     │  계약서에 대해 질문하기              [그리드 보기]   │           │
│ ────────  │  ──────────────────────────────────────────  │  출처 패널  │
│ ▸계약서 ●  │                                              │ (질문 후    │
│  약관     │   🙋 MFN 조항이 가장 유리한 계약은?               │  표시)     │
│  소송서면  │                                              │           │
│ ────────  │   🤖 ACME_MSA 계약이 가장 유리합니다. 18.2조에서… │ ┌───────┐ │
│ 대화기록   │      [출처 1: ACME_MSA p.12]                  │ │ACME p12│ │
│ ·오늘      │      [출처 2: Beta_NDA p.4]                   │ │인용문…  │ │
│  └ MFN..  │                                              │ └───────┘ │
│ ·어제      │  ┌────────────────────────────────────────┐  │           │
│  └ 지배법.. │  │ 계약서에 대해 무엇이든 물어보세요…       [↑]│  │           │
│           │  └────────────────────────────────────────┘  │           │
└──────────┴──────────────────────────────────────────────┴───────────┘
```

- **좌측**: 도메인(워크스페이스) 목록 — 유저는 **읽기 전용으로 선택만**. 하단에 대화 기록(세션). (현재 챗 히스토리는 비저장 → **세션 저장 필요**, §7 참고)
- **중앙 챗**: 기존 `ChatInterface.tsx` 확장. 답변에 **인라인 출처 칩**(문서명 + 페이지)을 붙여 클릭 시 우측 출처 패널/문서 뷰어로 연결. (현재는 출처 연결 없음 → 신규)
- **상단 `그리드 보기`**: 현재 선택 도메인의 읽기전용 그리드(U-1)로 전환.
- **빈 상태**: 추천 질문 칩 ("이 도메인에서 가장 흔한 조항은?", "계약일이 빠른 순서는?").
- **입력**: 멀티라인, 전송(↑). (스트리밍은 후속 단계)

### U-1. 검색 결과 (읽기전용 그리드)

일반 유저가 "그리드 보기"를 누르거나 검색했을 때. **A-1과 동일 레이아웃이나 편집 UI 전부 제거**.

```
┌──────────┬──────────────────────────────────────────────┬───────────┐
│ 도메인     │ 계약서   [🔍 검색…           ]        [챗으로]    │           │
│ ▸계약서 ●  │ ─────────────────────────────────────────────  │  문서 뷰어  │
│           │  문서명        │ 지배법   │ 계약일   │ MFN조항      │ (셀 클릭)  │
│           │ ──────────────┼────────┼────────┼──────────     │           │
│           │  ACME_MSA.pdf │ 뉴욕주   │ 2024-01 │ 있음          │           │
│           │  Beta_NDA.pdf │ 델라웨어 │ 2023-11 │ 없음          │           │
└──────────┴──────────────────────────────────────────────┴───────────┘
```

- A-1 그리드 재사용하되 **숨김**: 체크박스, `＋컬럼`, 컬럼 편집 진입, Run/Stop, 문서 추가/삭제, 검증 버튼.
- **추가**: 상단 검색바(문서명·셀값 필터, 후속 단계에서 의미검색으로 확장), 정렬/필터(읽기).
- 셀 클릭 → **읽기전용 인용 사이드바**(A-4에서 `검증 완료` 버튼만 제거한 형태) / 문서 뷰어(U-2).

### U-2. 문서 뷰어 / 인용 확인

챗 답변의 출처 칩 또는 그리드 셀에서 진입. 원문 마크다운 + 인용 하이라이트.

```
┌─────────────────────────────────────┐
│ ACME_MSA.pdf · p.12            [✕]   │
│ ───────────────────────────────────  │
│ … 18.2 Governing Law. This          │
│   Agreement shall be governed by    │
│   the laws of the State of          │
```

## 7. 기존 코드 대비 신규/변경 필요 (화면 관점)

기획을 구현으로 옮길 때 화면 측면에서 필요한 작업:

1. **워크스페이스 레이어 신규** — 현재는 단일 프로젝트(메모리). "도메인=문서종류" 다중 워크스페이스 + 좌측 레일 + 전환. (저장 모델은 후속 설계)
2. **역할 분기** — 같은 그리드/사이드바를 `readOnly`/`role` prop으로 편집 UI on/off. 라우팅은 제외했으므로 "모드 진입점"만 둘로.
3. **유저 챗 = 메인 레이아웃** — 챗을 사이드바가 아니라 **전체 화면 중앙**으로 승격(현재는 우측 400px 사이드바).
4. **챗 출처 연결** — 답변 → 문서/페이지 인용 칩 → 뷰어. (현재 없음. 챗 고도화 전이라도 화면 슬롯은 미리 잡아둠)
5. **챗 세션 저장** — 현재 unmount 시 소실. 대화 기록 목록(U-0 좌하단) 위해 필요.
6. **검색바** — U-1 상단. 초기엔 클라이언트 필터, 후속에 의미검색.
